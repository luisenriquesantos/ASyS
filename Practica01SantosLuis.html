Las gráficas integradas facilitan la visualización de los datos y la obtención de información a partir de ellos. M1.1 Inline Functions

Muchas funciones simples se representan más convenientemente mediante el uso MATLAB inline. Un objeto inline proporciona una representacion simbolica de una función definida en términos de operadores y funciones de MATLAB. Por ejemplo, considere la función:
f= @(t) exp(-t).*cos(2*pi*t);
Una vez definida la función podemos evaluarla en algun valor de interes. Por ejemplo,
t=0;
f(t)
ans = 1
Esto tambien se puede hacer avaluando directamente en la funcion:
f(0)
ans = 1
Las entradas vectoriales permiten la evaluación de múltiples valores simultáneamente. Considere la tarea de graficar f (t) durante el intervalo (−2 ≤ t ≤ 2).
t=(-2:2);
f(t)
ans = 1×5
7.3891 2.7183 1.0000 0.3679 0.1353
El comando plot representa gráficamente el resultado, que se muestra en la figura.
plot (t,f(t));
xlabel('t'); ylabel('f(t)'); grid;
Pero ¿cuántos puntos son suficientes? Si se eligen muy pocos puntos, se pierde información. Si se eligen demasiados puntos, se desperdician memoria y tiempo. Se necesita un equilibrio. Para las funciones, normalmente se representan de 20 a 200.
Para este ejercicio se elijen 100 puntos.
t=-2:0.01:2;
plot(t,f(t));
xlabel('t'); ylabel('f(t)'); grid;

M1.2 Relational Operators and the Unit Step Function

Con la ayuda de operadores relacionales, pueden representar la función de paso unitario.
En MATLAB, un operador relacional compara dos elementos. Si la comparación es verdadera, se devuelve un verdadero lógico (1). Si la comparación es falso, se devuelve un falso lógico (0).
Los operadores relacionales son:
<,>, <=,> =, == y ~ =.
La función de paso unitario se define fácilmente utilizando el operador relacional >=.
u = @(t) t>=0;
Cualquier función con una discontinuidad de salto, como la funcion de paso, es difícil de trazar. Considere trazar u (t) usando
t=-2:2;
plot (t,u(t));
xlabel ('t');ylabel('u(t)');
Dos problemas significativos son evidentes en la gráfica resultante. Primero, MATLAB escala automáticamente los ejes para ajustar los datos. En este caso, esta característica normalmente deseable oscurece la mayor parte de la trama. En segundo lugar, MATLAB conecta datos de trazado con líneas.
Considere trazar u (t) usando:
p=@(t) t>0 & t<1;
t=-1:0.01:2;
plot(t,p(t));
xlabel ('t'); ylabel('p(t)=u(t)-u(t-1)');
axis ([-1 2,-.1 1.1]);

Considere
g=@(t) exp(-t).*cos(2*pi*t).*(t>=0);
Una operación combinada de desplazamiento y escalado está representada por , donde a y b son constantes reales arbitrarias. Como ejemplo,

M1.4 Integracion Numerica y estimación de señal de energía.

Considere la señal La energía respectiva es
x=@(t) exp(-t).*((t>=0) & (t<1));
t=0:0.01:1;
El resultado final se obtiene mediante el comando:
Ex=sum(x(t).*x(t)*0.01)
Ex = 0.4367
Este resultado tiene un error del 1%. reduciendo el valor de , se puede hacer una mejor aproximación.
t=0:0.001:1;
Ex=sum(x(t).*x(t)*0.001)
Ex = 0.4328
Aunque es simple de visualizar, la aproximación rectangular no es la mejor técnica de integración numérica. La función MATLAB quad implementa una mejor técnica de integración numérica llamada cuadratura de Simpson.
Para operar, quad se requiere unafunción que describe el integrando, el límite inferior de integración y el límite superior de integración. Tenga en cuenta que no es necesario que Δt este especificado.
x_squered =@(t) exp(-2*t).*((t>=0)&(t<1));
Ex=quad(x_squered,0,1)
Ex = 0.4323
En este caso, el error relativo es de 0.0026%
Se pueden usar las mismas técnicas para estimar la energía de señales más complejas. Considere g (t), definido previamente. La energía es:
g_squared=@(t) exp(-2*t).*(cos(2*pi*t).^2);
t=0:0.001:100;
Eg=sum((g_squared(t))*0.001)
Eg = 0.2567
Una mejor aproximación:
Eg=quad(g_squared,0,100)
Eg = 0.2562
